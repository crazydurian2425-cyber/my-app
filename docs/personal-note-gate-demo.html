Make the traveler message required

A plan can't be submitted until the planner writes a real note to the traveler. Below: how it behaves + one choice for you.

1 · The rule

On Submit, the 「{旅行者}さんへのメッセージ」 / "Personal note to traveler" box must contain real text. The grey placeholder suggestion does not count — only what the planner actually types. Checked once at submit (it's a Review-step field), alongside the itinerary rules.

2 · What the planner sees (proposed: minimum ~10 characters)

Box is empty
Too short (e.g. "hi")
A real note → passes ✓

3 · One choice for you

How strict?

(A) Minimum ~10 characters — blocks blanks and one-word stubs like "hi"/"ok", but still very light. (my recommendation)
(B) Any non-empty text — even a single character passes (just can't be blank).
(C) A different minimum — tell me the number.

Either way the message is the same idea — just the threshold differs. Tell me A / B / C and "go", and I'll implement it.
