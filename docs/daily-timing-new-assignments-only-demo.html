Fix the daily timing — but only for the next planner, never the built ones

Existing and in-progress plans stay frozen. Only a fresh assignment picks up the corrected time.

The rule you asked for

  • Already built plans (approved / submitted) — untouched. Keep their current start time.
  • In-progress plans (a planner is building right now) — untouched. Their work is never re-timed.
  • Not-yet-assigned plans — corrected, so the next planner assigned starts with the right time.
Example: this traveler requested Late · 09:30, but it was mis-registered as Standard · 08:30. We want to correct it only going forward.

Try it — press “Apply fix”

Watch the three planners. The built one and the building one don’t move. Only the next assignment flips to 09:30.

Day 1 (arrival day) is exempt: for every planner, the arrival day always starts at the arrival time — this fix only affects the Day 2+ wake time.
