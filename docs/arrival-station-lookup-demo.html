到着地点の解決 — 交通手段で判定（現行 → 修正案）

The lookup already knows the arrival method and from — it just doesn't use them. The fix biases the query by method.

DEMO — nothing changed. Only the rail case changes (adds "Station" so it resolves to the station, not the city). Flight & car are unchanged.
① 鉄道で到着（姫路の例）   method: Shinkansenfrom: Kyotolocation: Himeji

✕ 現行

✓ 修正案

method=rail → append Station
query: "Himeji Station"
→ 姫路駅（ホテル隣接）

Flight already resolves correctly — the fix does NOT touch it (and never appends "Station").
③ 車・タクシーで到着   method: Carfrom: 近隣の街location: Himeji

≈ 現行

query: "Himeji, <city>"
→ 姫路市

✓ 修正案

method≠rail → do NOT force a station
query: "Himeji, <city>"
→ 姫路市（駅を誤って指定しない）
Car has no "station" — forcing 姫路駅 would be wrong. The fix only adds "Station" when the method is rail.
The rule (method‑aware):
• Flight / IATA code → "… Airport"  (unchanged)
• Rail (train / shinkansen / 新幹線 / JR / metro) → append "Station" so it resolves to the station, not the city — unless the name already says Station/駅
• Car / bus / other → leave as‑is (never force a station)
The arrival from (previous unit) stays as supporting context — an inter‑city rail leg from Kyoto confirms it's the station, not the city centroid.
