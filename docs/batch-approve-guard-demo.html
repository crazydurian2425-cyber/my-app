🔧 What the guard actually changes (one function)

  1. In approveFullSet(setId, plannerId), before flipping the assignment to completed, count that planner's plans where status is not yet approved.
  2. If any are still submitted / in progress / pending, stop and show: "3 plans still need review — approve them before closing the batch." The assignment stays open.
  3. Only when every plan is approved does the button proceed and mark the batch completed / awaiting payout.
  4. Net effect: a plan submitted after your last review pass (Plan 5's exact case) can never be silently dropped from a planner's pay again.
