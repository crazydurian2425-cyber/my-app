PART 2 Copy the database · 1–2 hrs · copy-paste commands

Where In BOTH Supabase projects: Project Settings → Database → Connection string → choose URI. Copy each into your notepad and label them OLD and NEW. Replace [YOUR-PASSWORD] inside each string with that project's database password.
Where PowerShell (press Start, type "powershell"). Install the Postgres tools (one time only): 
winget install PostgreSQL.PostgreSQL
Close PowerShell and open a new one after install finishes.
Export the OLD database — run these 3 commands one at a time (replace OLD_CONNECTION_STRING with the real one, keep the quotes): 
npx supabase db dump --db-url "OLD_CONNECTION_STRING" -f roles.sql --role-only
npx supabase db dump --db-url "OLD_CONNECTION_STRING" -f schema.sql
npx supabase db dump --db-url "OLD_CONNECTION_STRING" -f data.sql --use-copy --data-only
This only READS the old database — it cannot change it.
Import into the NEW database — one command (replace NEW_CONNECTION_STRING): 
psql --single-transaction --variable ON_ERROR_STOP=1 --file roles.sql --file schema.sql --command "SET session_replication_role = replica" --file data.sql --dbname "NEW_CONNECTION_STRING"
⚠️ Triple-check this one uses the NEW string. If it errors, copy the error text and ask Claude — do not retry blindly.
Check it worked: NEW project → Table Editor → open travelers → you should see your 133 travelers with the December dates.
Turn on realtime: NEW project → Database → Replication → under supabase_realtime enable the same tables as the old project (at minimum messages).
