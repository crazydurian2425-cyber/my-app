Auto‑time · stops stay where you put them

Demo of the proposed behavior. The start time is read‑only (notice the 🔒) — it always auto‑computes. You only set the order (add buttons) and the stay length (dropdown).

Rules shown here:
1 · Hotel check‑in is pinned first (luggage drop) — nothing can move above it.
2 · A new activity always lands at the bottom and never jumps up.
3 · Each start = previous finish + travel, rounded UP to the nearest 5 min (5:46 → 5:50, never 5:45).
4 · Change a stay → only the times below shift. Order never changes.
Try it: change a “stay” dropdown and watch every time below recompute (still 5‑min rounded). Press + Add activity — it appends at the bottom with an auto time. There is no way to type a time, so nothing can leapfrog the hotel check‑in.
