What the real build does

  1. New table terms_versions (or one row with two JSON blobs) holding the Normal and Premium content — both seeded identical to today's terms, so nothing changes until you edit.
  2. A Terms editor screen in superadmin999 (the "CONTENT · Terms editor" nav) to edit either version.
  3. One column on planners: terms_variant, default 'standard' — every existing and new planner sees Normal unless you move them to Premium.
  4. terms.html keeps its single title (利用規約 / Terms of Service) and loads the body for the logged-in planner's variant from their session — can't be changed via the URL.
