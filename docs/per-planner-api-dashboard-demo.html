How Option A works (what this demo represents): a tiny counter wraps each Google call in the planner's browser and batches an increment to a new api_usage_daily table (planner · date · api · count) every ~30s. This admin page reads that table live. Effort ≈ 1 day, low risk — it only ADDS a counter; it never changes how Google is called, so a bug here can't break lookups. Pairs with the Google Cloud quota cap (this page SHOWS a runaway planner; the cap STOPS one).
