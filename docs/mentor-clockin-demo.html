Mentor clock-in — registered laptop, only 09:15–09:30 IDEA DEMO · nothing built

Three simple checks, all done on the server: (1) the account is a mentor, (2) it's the mentor's registered laptop, and (3) the time is between 09:15 and 09:30. No phone, no fingerprint, nothing to install. Click through the laptop below.

FREE

No new cost. A saved device key marks the registered laptop, and the time check runs on the server — both on tools you already pay for (Supabase + your Cloudflare Worker). Nothing extra to buy, nothing for the mentor to set up.

What the checks stop

What it blocks

✓
Clocking in from home / another PC — only the one registered laptop works.
✓
Early or late clock-in — before 09:15 or after 09:30 the server refuses. The laptop's own clock can't fool it.
✓
Faking the time — the time is read on the server (Asia/Tokyo), not the laptop.

Note: this page is a mock-up to explain the idea — it is not wired to your database and nothing was added to the app. Building it for real = one small “mentor devices” table + a “clock-in” endpoint on your Cloudflare Worker that checks the device key and the 09:15–09:30 Tokyo time, writing to a mentor_attendance table.
