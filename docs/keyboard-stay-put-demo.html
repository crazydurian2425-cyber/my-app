The search box stays exactly where it was — suggestions open right below it, the keyboard slides up from the bottom, nothing else moves. Only if a field would be hidden under the keyboard does the sheet content scroll — and only the minimum amount.

🔧 What would change in the real page (implemented only after you approve)

  1. NowWhen the keyboard opens, a handler force-scrolls the focused field to the center of the screen (added in the previous cursor fix) — AND iOS pushes the whole page up on its own. Two movements stacked = the jumpy mess.
  2. AfterRemove the force-center scroll. Instead, only when a field would actually be hidden under the keyboard, nudge the sheet's inner scroll the minimum amount ("nearest"). A field that's already visible → nothing moves at all.
  3. AfterPin the sheet's bottom edge to the top of the keyboard. Right now the sheet stays anchored to the bottom of the screen (behind the keyboard), so iOS decides the input is hidden and yanks the whole page up. If the sheet always fits above the keyboard, iOS has no reason to move anything.
  4. The earlier cursor-drift fix stays — with the screen no longer moving, the root cause of the drift disappears too.
