The change: when a hotel check‑out is added, time it by its position in the day (the same time‑chain every other stop uses) instead of hard‑copying the hotel's 10:45. So a check‑out placed first lands at 8:30 / 8:40, and the planner can schedule an early checkout + later departure.

The hotel's 10:45 stays useful as the latest allowed checkout (a cap/hint) — you just won't be forced to it. And a non‑hotel first activity already behaves this way; this makes the check‑out consistent with it.
